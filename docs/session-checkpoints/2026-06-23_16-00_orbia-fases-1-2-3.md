## Objetivo Original

Construir el juego móvil Orbia desde cero siguiendo el plan de 9 fases documentado en `orbia-plan-desarrollo.md`. Stack: Phaser 3 + Matter.js + Capacitor. Todo generado en canvas, sin assets externos.

## Contexto del Proyecto

- Juego de física: pantalla llena de esferas numéricas que rebotan. El jugador toca la pantalla para crear un punto de desvío temporal que redirige trayectorias.
- Objetivo: hacer colisionar esferas del mismo número (100 pts) o mismo número + color (250 pts).
- Servidor de desarrollo corriendo en `http://localhost:5173` con Vite.

## Decisiones Tomadas

1. **Vite en lugar de http-server**: más rápido y con HMR para desarrollo.
2. **frictionAir: 0 + restitution: 1.0**: las esferas nunca paran; se mantiene velocidad mínima en update().
3. **Velocidad mínima en update()**: MIN_SPEED=2, MAX_SPEED=5 para control de juego.
4. **Canvas centrado con CSS absolute + transform**: soluciona el canvas retrato en pantalla apaisada de escritorio.
5. **Punto de desvío como cuerpo estático Matter.js** con restitution: 1.2 para rebotar con más fuerza.
